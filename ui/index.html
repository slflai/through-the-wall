教程 · Through the Wall

一個幫你從 IG、Pinterest、Twitter 等平台保存圖片、影片、Reels、Stories 的本地 app。

快速開始

  1. 在「儲存」分頁貼上任何社群貼文 URL
  2. 按「抓取」，app 會列出所有可下載的媒體（carousel 會全部列出來）
  3. 勾選想要的項目、視需要改檔名
  4. 填分類（例如 reference/fashion，可以巢狀），按「儲存」
  5. 檔案會存到 ~/Desktop/through-the-wall/<分類>/

支援的平台

  • Instagram：公開貼文、carousel、Reels、Stories（你已登入的私密帳號也可以）
  • Pinterest：圖片 pin、影片 pin、GIF
  • Twitter / X：貼文裡的圖片與影片
  • TikTok / YouTube / Vimeo：影片
  • 其他幾百個站（底層使用 gallery-dl + yt-dlp）

Cookie / 登入內容

需要登入才看得到的內容（私密帳號、Stories、限動）會自動從你 Chrome 的登入狀態讀取 cookie。

第一次使用 macOS 會跳出「允許存取鑰匙圈」的提示，按允許一次就好。

圖庫管理

  • 點一下檔案：在 Finder 顯示
  • 右鍵檔案：重新命名、移動到其他分類、刪除
  • 右鍵資料夾：重新命名、移動、刪除、新增子分類
  • 左上角 ＋ 按鈕：新增根分類

常見問題

抓取失敗怎麼辦？

先確認你在 Chrome 裡有登入該平台（尤其是私密內容）。某些 IG 貼文需要頻繁重新登入或平台臨時限流。換個網路/等一下重試通常會好。

影片只有聲音沒畫面？

已在 v0.2 修正；確保 yt-dlp 優先選擇含影像的 format。若仍發生，請回報該 URL。

想改儲存位置？

到「設定」分頁修改。也可以把現有的 ~/Desktop/through-the-wall 資料夾直接搬到別處再更新設定即可。

開源 / 授權

這個專案打算以 MIT License 開源。核心技術堆疊：

  • yt-dlp — 影片下載
  • gallery-dl — 圖片 / 圖庫下載
  • pywebview — 原生視窗

設定

預設 ~/Desktop/through-the-wall。每個分類會是一個子資料夾。
自動從你登入的 Chrome 讀取，不需額外設定。macOS 第一次可能會要求授權鑰匙圈。
